# GAPS (Graph-based Automated Path Synthesizer)

GAPS is a hybrid analysis tool designed for Android application testing. It features two primary running modes:
1. **Static Analysis (`static`)**: Produces call-sequences and high-level instructions for automatic interaction.
2. **Dynamic Analysis (`run`)**: An automatic interaction module that uses the high-level instructions generated by the static analysis to interact with the app dynamically.

---

## 🛠 Prerequisites

Before running GAPS, ensure you have the following installed:

1. **[uv](https://docs.astral.sh/uv/)**: Fast Python package installer and resolver.
2. **`apktool`**: Must be accessible in your `PATH`. [Download apktool](https://apktool.org/).
3. **`baksmali`**: (Optional) Required only if you are interested in DEX-only analysis.
4. **Rooted Android Device/Emulator**: (Optional) Equipped with Frida if you want to obtain Runtime Reachability during the `run` mode.

## 🚀 Installation

GAPS uses `uv` for lightning-fast dependency management and environment isolation.

1. **Install `uv`** (if you haven't already):
   ```bash
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```
   *(For macOS/Windows or pipx instructions, see [uv installation guide](https://docs.astral.sh/uv/getting-started/installation/))*

2. **Setup the project**:
   Navigate to the GAPS directory and sync the dependencies. `uv sync` will automatically create a `.venv` virtual environment and install the package.
   ```bash
   uv sync
   ```

## 📖 Usage

### 1. Static Analysis Mode
The `static` mode disassembles the application and generates the paths and instructions needed for the dynamic run.

#### Command Line Arguments (`static`):
- `-i, --input PATH` : APK/DEX path file to disassemble **[required]**
- `-m, --method TEXT` : Target method to generate paths from
- `-cls, --class_name TEXT` : Target class to generate paths from
- `-p_cls, --parent_class TEXT` : Find target method invocation in a specific parent class
- `-sig, --signature TEXT` : Target method signature in Smali format to build paths for it
- `-seed, --seed_file PATH` : Path to the file containing seed signatures
- `-custom_seed, --custom_seed_file PATH` : Path to the file containing custom seed signatures
- `-cond, --conditional` : Generate paths that satisfy conditional statements
- `-l, --path_limit INTEGER` : Set an upper bound to the total number of paths reconstructed for each query (default: 1000)
- `-up, --unconstrained-paths`: Generate paths without constraints
- `-v, --verbose` : Enable verbose output
- `-d, --debug` : Enable debug output
- `--help` : Show this message and exit

**Example Usage**:
```bash
uv run gaps static -i <app-path> -cond -m <target-method> -o <output_dir>
```
*Note: If no search direction is provided (target method, class, or signature), GAPS will automatically generate a seed file by randomly selecting 50 methods from the app's package.*

### 2. Dynamic Automatic Interaction Mode
The `run` mode takes the instructions generated in the `static` step and automatically explores the app. By default it uses a built-in LLM agent to interact with the hierarchy dynamically.

#### Command Line Arguments (`run`):
- `-i, --input PATH` : APK path file to run **[required]**
- `-instr, --instructions PATH` : JSON instruction file generated from `static` mode
- `-frida, --frida` : Add Frida hooks (requires rooted device/emulator with frida-server running)
- `-ms, --manual-setup` : Pause execution to let you manually set up the app (e.g., login) in the initial stage
- `-t, --target TEXT` : Specify a specific target method from the instructions file to reach
- `--help` : Show this message and exit

**Example Usage**:
```bash
# Don't forget to export your LLM API Key (if required by the LLM agent)!
export OPENAI_API_KEY="your-api-key"

uv run gaps run -i <app_path> -instr <path_to_json> -frida
```

## 📊 Paper Evaluation Data & Benchmarks

The `evaluation_data` directory contains the results of the experiments described in the research paper, along with the target methods considered.

### Running the AndroTest Benchmark
The AndroTest benchmark dataset instrumented by AndroLogs can be downloaded from [here](https://drive.google.com/file/d/16RnqK_L0I90e2OzqjDM9YMK5PKvnILHo/view?usp=sharing).

1. Download and unzip the dataset into a directory.
2. Run the static analysis on the dataset:
   ```bash
   cd scripts
   ./run_all_static.sh <androtest_directory_path> ../evaluation_data/androtest_seeds <output_directory>
   ```
   *This will generate a directory for each app containing high-level instructions, logs, and a `stats.csv` file.*

3. Plug in an Android device or start an emulator, then run the dynamic analysis:
   ```bash
   ./run_all_dynamic.sh <androtest_directory_path> <output_directory>
   ```

4. Generate refined statistics:
   ```bash
   cd stats # while in the scripts directory
   python get_stats.py ../<output_directory>/stats.csv <androtest_directory_path> ../../evaluation_data/androtest_seeds
   ```
   This process will output a `final_stats.csv` file inside your `<output_directory>`.
